Project Template Guide

A simple step-by-step guide for creating and publishing a project page in the Simulation & Modelling Society website.

What is this page for?

Every project in our society has its own web page. This page is like a “profile” for your project.

It contains links to your code, reports, notes, and final release — so anyone can understand what your project is and how it works.

What you will create

1. A project webpage

A simple HTML file that describes your simulation project.

2. A GitHub repository

Where your actual code lives and is updated.

3. Documentation files

Reports, math notes, and development logs (PDF or Markdown).

How the system works

Think of your project like a folder with two parts:

  • Code (GitHub): The actual simulation program
  • Website page: The explanation and documentation of that project

The project template connects everything together in one place.

Step-by-step: How to use the template

Step 1 — Clone the Official Repository

Goal: Get the official society system onto your computer.

Why: All projects must be created inside the official repository to be tracked and reviewed.

What to do:

git clone https://github.com/mcsms-org/mcsms-org.github.io.git
cd mcsms-org.github.io

Expect: You now have a local working copy of the society website.

Avoid: Do NOT download ZIP files — always use Git clone.

Step 2 — Create a New Branch

Goal: Create a safe workspace for your project.

Why: Prevents breaking the main website while you develop.

What to do:

git checkout -b project-your-name

Expect: You are now working inside a separate branch.

Avoid: Never edit main directly.

Step 3 — Create Project Folder from Template

Goal: Create a structured project workspace.

What to do:

  1. Copy templates/project-ex.html
  2. Paste into projects/project-your-name/
  3. Rename main file to index.html

Structure:

projects/project-your-name/
    index.html
    simulation-report.pdf
    technical-report.pdf
    math-notes.pdf
    dev-log.md
    release-notes.md
    assets/

Expect: A self-contained project directory.

Step 4 — Fill Project Content

Goal: Document your simulation properly.

What to do: Edit index.html:

  • Project title & abstract (2–3 lines)
  • Tags (Physics / Math / Systems etc.)
  • GitHub repository link
  • Report links (PDFs)
  • Dev logs (Markdown)
  • Release notes & Contributors

Avoid: Leaving placeholder links (#) in the final version.

Step 5 — Commit and Push

Goal: Upload your project to the online repository.

What to do:

git add .
git commit -m "Add project: your project name"
git push origin project-your-name

Expect: Your project exists on GitHub under your branch.

Avoid: Forgetting to push (project stays local).

Step 6 — Create Pull Request (PR)

Goal: Submit your project for review.

Why: This is the official “publication gate” of the society.

What to do: On GitHub, click "Compare & Pull Request". Describe what the simulation does, files included, and key results.

Expect: Your project enters the review stage.

Step 7 — Register Project on Website

Goal: Make your project visible on the public Projects page.

What to do: Open projects/projects-index.json and add your new project object to the list:

{
  "id": "your-project-name",
  "title": "Your Project Title",
  "description": "Short description of your simulation.",
  "tags": ["Tag1", "Tag2"],
  "link": "projects/your-project-name/"
}

Expect: Your project will appear automatically on the Projects page.

Step 1 / 7

Important rules

  • Every project must have a GitHub repository
  • Every project must have at least one report (Simulation Report)
  • Every project must have a version number (example: v1.0.0)
  • Documentation must explain what the simulation does and what it shows

Why we do it this way

This system is designed so that every project is:

  • Easy to understand for new members
  • Easy to share with others
  • Properly documented like real scientific work

It also helps you learn how real engineering and research projects are structured.

HELP

Still confused?

If anything feels unclear, don’t worry — this system is meant to be learned gradually while building your first project.
